# RTLign Project Structure

## Root Directory

```
RTLign/
├── orchestration/          # Pipeline orchestration scripts
├── ml_predictor/           # ML models and parsing utilities
├── rtl_legalizer/          # Verilog RTL legalizer hardware
├── openroad_scripts/       # OpenROAD DEF files and TCL scripts
├── data/                   # Benchmarks and training data (gitignored)
└── .kiro/                  # Kiro specs and steering files
```

## Module Organization

### orchestration/
Master pipeline control and automation.

- `master_run.py` - Single-click pipeline orchestrator (4 stages: LEF parse → DEF parse → RTL simulate → HEX inject)

### ml_predictor/
Parsers, ML models, and coordinate prediction.

- `def_parser.py` - Extract macro coordinates from DEF files → `.hex` format
- `hex_to_def.py` - Inject legalized coordinates back into DEF files
- `lef_parser.py` - Extract real cell dimensions from LEF library files

**Planned files:**
- `feature_extractor.py` - Extract placement features from DEF
- `train_rf.py` - Train Random Forest predictor
- `train_nn.py` - Train neural network predictor
- `predict.py` - Generate predicted placements

### rtl_legalizer/
Verilog RTL legalizer hardware.

- `collision_check.v` - Combinational AABB overlap detector
- `legalizer_fsm.v` - FSM-based greedy sweep legalizer with die-boundary clamping
- `legalizer_tb.v` - Testbench with overlap audit and VCD dump
- `dummy_layout.hex` - Input memory file (generated by def_parser)
- `output_layout.hex` - Legalized output (generated by testbench)
- `sim.out` - Compiled Verilog simulation binary
- `legalizer.vcd` - Waveform dump for debugging

**Key parameters:**
- `NUM_LINES` = 672 (168 macros × 4 fields)
- `DIE_WIDTH` = 200260, `DIE_HEIGHT` = 201600 (database units)

### openroad_scripts/
OpenROAD physical design files.

- `mockup_export.def` - Baseline GCD design (482 components, 168 placed macros)
- `legalized_export.def` - Output: legalized placement

### data/
Training datasets and benchmarks (gitignored, shared via Google Drive).

```
data/
├── rtl_sources/           # RTL designs (PicoRV32, Ibex, SweRV, OpenTitan)
├── ispd_benchmarks/
│   ├── ispd2011/          # Bookshelf format (8 designs)
│   ├── ispd2015/          # DEF + LEF format (18 designs, 28K-1.3M components)
│   └── ispd2023/          # DEF + ASAP7 (6 crypto designs)
├── openroad_configs/      # Per-design OpenROAD configs (planned)
├── generated_defs/        # Output from automated OpenROAD runs
└── training_ready/        # ML features extracted from DEFs
```

### .kiro/
Spec-driven development artifacts.

```
.kiro/
├── specs/
│   └── lef-parser-real-dimensions/  # Current spec in progress
│       ├── requirements.md
│       ├── design.md
│       └── tasks.md
└── steering/
    ├── product.md         # Product overview
    ├── tech.md            # Tech stack and commands
    └── structure.md       # This file
```

## Data Flow

```
DEF → def_parser.py → HEX → legalizer_fsm.v → HEX → hex_to_def.py → DEF
         ↑                                           ↓
    dimension_dict                              OpenROAD GUI
         ↑
    lef_parser.py
         ↑
    LEF files
```

## File Naming Conventions

- Python: `snake_case.py`
- Verilog: `snake_case.v`
- Data files: `snake_case.def`, `snake_case.lef`, `snake_case.hex`
- Documentation: `UPPERCASE.md`, `README.md`

## Output Files (Generated)

These files are created by the pipeline and should not be edited manually:

- `rtl_legalizer/dummy_layout.hex` - Input coordinates for legalizer
- `rtl_legalizer/output_layout.hex` - Legalized coordinates
- `rtl_legalizer/sim.out` - Compiled simulation binary
- `rtl_legalizer/legalizer.vcd` - Waveform dump
- `openroad_scripts/legalized_export.def` - Final output DEF

## Benchmark Data Structure

ISPD 2015 benchmark directories contain:
```
mgc_matrix_mult_2/
├── tech.lef              # Technology library (DATABASE MICRONS, layers)
├── cells.lef             # Cell library (MACRO definitions with SIZE)
├── floorplan.def         # Placed design (COMPONENTS section)
├── design.v              # Verilog netlist
└── placement.constraints # Placement constraints
```
